# Output Format and Actions — clarify-ticket

## Output format

Present the analysis in terminal using this structure:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔍 TICKET ANALYSIS: {key} — {title}
   Status: {status} | Type: {type} | Points: {points}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 ASSESSMENT: {one of: ✅ Ready | ⚠️ Needs clarification | 🚫 Has blockers}

{One paragraph summary of overall ticket health}

━━ BLOCKERS ({count}) ━━━━━━━━━━━━━━━━━━━━━━━━

1. [{audience}] {clear actionable question}
   ↳ Context: "{quote from ticket that triggered this}"

2. [{audience}] {clear actionable question}
   ↳ Context: "{quote from ticket that triggered this}"

━━ NICE-TO-HAVE ({count}) ━━━━━━━━━━━━━━━━━━━━

1. [{audience}] {question or suggestion}
2. [{audience}] {question or suggestion}

━━ STRUCTURAL CHECK ━━━━━━━━━━━━━━━━━━━━━━━━━━

  ✅ Description present
  ✅ Acceptance criteria defined
  ❌ No design link found
  ❌ No estimation
  ✅ Assignee: {name}
  ⚠️ Type mismatch: labeled as Story but reads like a Task

━━ UNANSWERED QUESTIONS ━━━━━━━━━━━━━━━━━━━━━━

From comments:
- {date} — {author}: "{question}" → NO RESPONSE FOUND

━━ CONTRADICTIONS ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Description says "{X}" but comment from {author} on {date} says "{Y}"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## Actions

After displaying the report, offer these actions:

1. **Comment on ticket** — Post the blockers and nice-to-haves as a formatted comment using the adapter's comment tool. Use this format for the comment body:

```
**Ticket Analysis — Automated Review**

**Blockers ({count}):**
1. [{audience}] {question}
2. [{audience}] {question}

**Suggestions ({count}):**
1. [{audience}] {question}

**Structural Issues:**
- {issue}

_Generated by refinery/clarify-ticket_
```

2. **Update ticket description** — Append a "Open Questions" section to the existing description using the adapter's edit tool. Never replace the original description.

3. **Copy to clipboard** — Display as-is, no tool calls.

4. **Re-analyze with context** — Ask the user for additional context, then re-run the analysis incorporating their input.
